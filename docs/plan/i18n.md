# 国际化（i18n）计划

## 背景

当前前端所有用户可见文本均为英文硬编码，用户群体以中文为主（代码中已出现中文字符串）。本计划目标是将 web 前端完整国际化，优先支持中文（简体），预留其他语言扩展能力。

---

## 技术选型

**库：`i18next` + `preact-i18next`**

理由：
- 生态成熟，社区最大
- 支持 Preact（通过 react 适配层）
- 按需加载语言包（懒加载）
- 支持插值、复数、嵌套 key
- 不依赖构建工具，Vite 开箱即用

**替代方案：`@solid-primitives/i18n`（太轻，功能不足）、`lingui`（需 babel 插件，构建复杂）**

---

## 支持语言

| 优先级 | 语言 | locale |
|--------|------|--------|
| P0 | 英文（默认） | `en` |
| P0 | 简体中文 | `zh-CN` |
| P1 | 繁体中文 | `zh-TW` |
| P2 | 日文 | `ja` |

语言检测顺序：用户手动设置 → localStorage → 浏览器 `navigator.language` → 降级 `en`

## 特殊处理事项

### 不翻译的内容
- Emoji 图标（●、○、↺、✎、✕、↵、⌫）
- 快捷键符号本身（Ctrl、Esc、Tab —— 这些是通用符号）
- 会话名称、项目名称（用户数据）
- 后端返回的错误消息（后端目前无 i18n，暂时原样展示）
- Agent 类型名称（`claude-code`、`codex`、`opencode` —— 产品名）

### 复数处理
- `"{n} active session"` / `"{n} active sessions"` → 使用 i18next 的 `_one` / `_other` 后缀
- 中文不区分单复数，直接写 `{n} 个活跃会话`

### 日期时间
- 现有代码使用 `toLocaleDateString()` / `toLocaleTimeString()`，默认跟随系统 locale，不受应用语言切换影响
- 需统一封装一个 `formatDate(value, locale)` helper，所有日期时间调用点都传入 `i18n.language`，确保语言切换后时间格式同步变化

### 语言切换 UI
- 在侧边栏底部或设置页加语言下拉菜单
- 切换时调用 `i18n.changeLanguage()` 即时生效，同时写入 localStorage 持久化
- 无需刷新页面

## 实施阶段

### Phase 1 — 基础设施
- 安装 `i18next`、`preact-i18next`、`i18next-parser` 依赖
- 配置 i18next 初始化
- 在 `web/package.json` 增加脚本：`i18n:extract`（提取 key）、`i18n:check`（检测 missing key，返回非零退出码）
- 在 CI（`.github/workflows/ci.yml`）增加 `i18n:check` gate，missing key 即 fail
- **基线扫描**：扫描 `web/src/` 所有用户可见字符串，生成完整待翻译清单（包含 `SubSessionBar.tsx`、`SubSessionCard.tsx`、`SubSessionWindow.tsx` 等子会话组件），清单覆盖率须达 100% 才可进入 Phase 2
- 建立 `en.json` 基准文件
- 建立 TypeScript key 类型定义，确保编译期 key 检查

### Phase 2 — 组件替换（高优先级文件）
- 逐文件将硬编码字符串替换为 `t('key')` 调用
- **全量替换 `toLocaleDateString()` / `toLocaleTimeString()` 为 `formatDate(value, i18n.language)` helper**（Phase 2 内完成，不留到后续）
- 同步完成 `zh-CN.json` 翻译

### Phase 3 — 组件替换（中优先级文件）
- 剩余组件替换，含 `SubSessionBar.tsx`、`SubSessionCard.tsx`、`SubSessionWindow.tsx`
- 处理复数、插值等复杂场景
- 验收时额外检查：无直接 `toLocale*` 调用残留（grep 确认）

### Phase 4 — 语言切换 UI
- 实现语言选择组件
- 语言偏好持久化

### Phase 5 — 扩展语言
- 根据需要添加 `zh-TW`、`ja` 等

## 验收标准

每个 Phase 完成后需满足以下条件才可合并：

| 检查项 | 方式 |
|--------|------|
| 无 missing key | `npm run i18n:check` 返回 0，CI gate 通过 |
| en/zh-CN 切换即时生效 | 手动切换语言，无需刷新，文本立即变化 |
| 未翻译 key 回退到 en | 删除 zh-CN.json 中某 key，确认显示英文而非 key 字符串 |
| 复数在 en 正确 | `_one`/`_other` 边界值验证（0、1、2） |
| 日期时间随语言切换 | 切换语言后时间格式变化（如 MM/DD/YYYY vs YYYY年MM月DD日） |
| 语言偏好持久化 | 切换后刷新页面，语言设置保留 |
